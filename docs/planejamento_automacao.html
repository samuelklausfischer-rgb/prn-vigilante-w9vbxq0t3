📋 O Que É Este Projeto?

Um sistema inteligente que envia lembretes de consultas médicas via WhatsApp de forma automatizada, mas com comportamento humanizado para evitar bloqueios.

🏥 Contexto

Clínica médica que precisa enviar lembretes de consultas para pacientes via WhatsApp

📱 Tecnologia

Evolution API (não-oficial) + Supabase (banco) + TypeScript

⚖️ Desafio

Enviar mensagens sem parecer robô para não ser banido pelo WhatsApp

🔒 Conformidade

Respeitar LGPD (consentimento, opt-out, retenção de dados)

🔄 Como Funciona? (Fluxo Principal)

1

📥 Recebe Mensagens

O sistema busca mensagens na fila com status "queued", aprovadas e agendadas para agora

⬇️
2

🔒 Trava a Mensagem

Usa "SKIP LOCKED" para garantir que só um worker processe cada mensagem (sem duplicar)

⬇️
3

🎯 Seleciona Instância

Escolhe qual WhatsApp usar (round-robin: um depois do outro, distribuindo carga)

⬇️
4

⏱️ Aguarda Delay

Espera tempo aleatório (3-12 minutos) desde o último envio daquela instância

⬇️
5

📤 Envia Mensagem

Chama Evolution API para enviar. Se falhar, tenta novamente (até 3x)

⬇️
6

✅ Atualiza Status

Marca como "delivered" no banco e registra métricas. Se falhou, marca como "failed"

🔄 Queries do Banco ALTA

Funções SQL otimizadas para concorrência

  • claim_next_message (selecionar próxima)
  • update_message_status (atualizar)
  • release_expired_locks (liberar locks)

🔌 Evolution Driver ALTA

Cliente para comunicação com Evolution API

  • sendTextMessage (enviar texto)
  • getConnectionStatus (verificar status)
  • Retry com backoff exponencial
  • Rate limiting

📋 Queue Manager ALTA

Gerenciamento da fila de mensagens

  • Buscar mensagens disponíveis
  • Lock atômico (SKIP LOCKED)
  • Marcar como enviada/falhou

🎯 Instance Selector ALTA

Escolher qual WhatsApp usar

  • Round-robin simples
  • Load-based (menor carga primeiro)
  • Health check

🤖 Worker Engine ALTA

Motor principal que roda continuamente

  • Loop principal
  • Graceful shutdown
  • Heartbeat (sinal de vida)
  • Orquestração

🔄 Retry Manager MÉDIA

Tentar novamente em caso de falha

  • Backoff exponencial (1s, 2s, 4s, 8s...)
  • Máximo 3 tentativas
  • Apenas erros recuperáveis

⚡ Circuit Breaker MÉDIA

Proteção contra falhas em cascata

  • Fechar após 5 falhas
  • Reabrir após 1 minuto
  • Evita sobrecarga

📝 Logger Estruturado MÉDIA

Logs em JSON para monitoramento

  • Níveis: trace, debug, info, warn, error
  • Sanitização LGPD (anonimizar telefones)
  • Correlation ID

📊 Metrics Collector MÉDIA

Métricas para dashboard

  • Taxa de sucesso
  • Tempo médio de entrega
  • Mensagens por hora
  • Exportar Prometheus

⏱️ Anti-Ban Delay BAIXA

Delay aleatório para parecer humano

  • 3-12 minutos aleatórios
  • Por instância
  • Respeita último envio

🎭 Text Spinning BAIXA

Variar texto para não repetir

  • Templates com sinônimos
  • "Olá" vs "Oi" vs "Bom dia"
  • Rotação de variações

✂️ Message Splitter BAIXA

Dividir mensagens longas

  • Quebrar em 1000-1500 caracteres
  • Respeitar parágrafos
  • Delay entre partes
