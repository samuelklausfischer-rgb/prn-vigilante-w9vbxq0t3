Visão Geral do Sistema

O projeto é um MVP/aplicação robusta de automação de disparo de mensagens do WhatsApp (Squad "Sara") baseada em filas. Focado para a empresa PRN, eliminando entrada manual e banimentos.

React + Vite Supabase Tailwind css Evolution API

Regras de Governança (Strict Rules)

  • Regra #1 (Doc-Sync): Alterou código .ts/tsx? É OBRIGATÓRIO atualizar a documentação de arquitetura na pasta docs/ correspondentemente.
  • Regra #2 (Propagação): Alterou o Supabase schema? Modelos em types/ e requisições em services/ precisam ser verificados pelo sub-agente.
  • Regra #3 (Status Visual): Sempre usar este Dashboard para consulta antes de modificar fluxos core.

Mapeamento de Módulos (Diretórios)

/src/pages

  • Index.tsx (QueueList): Painel de aprovação das filas pendentes de disparo da Sara.
  • WhatsAppSettings.tsx: Monitoramento e criação de sub-workers (instâncias pareadas por QR Code).
  • Auth.tsx: View login (proteção via RLS do Supabase).

/src/services

  • evolution.ts: Controla as requisições de criação e deleção de instâncias usando edge functions do Supabase.
  • data.ts: Fetch de Patients Queue (com Mock de Fallback) e Update de orders draggables.

/supabase

  • migrations/: Scripts de banco de dados (`init_schema`, tabelas para `whatsapp_instances` e filas).
  • functions/: Ponto de entrada de proxy do Supabase para o Webhook da máquina local / cloud da Evolution.
