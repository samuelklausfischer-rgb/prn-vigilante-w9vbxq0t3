📩 Fluxo de Mensagem Passo-a-Passo

Como uma mensagem viaja do Dashboard até o WhatsApp

1 📝 Passo 1: Usuário adiciona paciente

No Dashboard React, usuário insere nome, telefone, data do exame, procedimentos

POST → supabase.functions.create-enqueue-patient

2 🔍 Passo 2: Enfileiramento com deduplicação

Function enfileira paciente e verifica se não existe duplicata recente (< 2h)

INSERT INTO patients_queue
WHERE canonical_phone + data_exame + horario NOT IN (últimas 2h)

3 ⏳ Passo 3: Fila aguarda envio

Mensagem fica em status 'queued' até horário de envio (send_after)

status: queued send_after: 2025-03-21 14:00

4 🤖 Passo 4: Worker claima mensagem

Worker gera lease, executa a cada 5s, claima próxima mensagem elegível

SELECT * FROM claim_next_message(p_worker_id, p_max_attempts)
WHERE status = 'queued' AND send_after <= NOW()

5 🧠 Passo 5: Humanização Anti-Ban

Simula digitação humana + delay aleatório para evitar detecção WhatsApp

⏱️ Simulando Digitação:
300 caracteres / 6 CPS = 50 segundos
⏳ Delay Anti-Ban:
3 a 13 minutos (randomizado)

6 📤 Passo 6: Envio via Evolution API

Envia mensagem para Evolution API com retry (3 tentativas exponenciais)

POST /message/sendText/{instance}
{ number: "5571912345678", text: "Olá Sr(a) João..." }
Retry: 3 tentativas com backoff exponencial (até 6s total)

7 ✅ Passo 7: Confirmação e Follow-up

Mensagem marcada como 'delivered', agendado follow-up em 1h se não houver resposta

✅ delivered_at: 2025-03-21 14:05
⏰ followup_due_at: 2025-03-21 15:05
